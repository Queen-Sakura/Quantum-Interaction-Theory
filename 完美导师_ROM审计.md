# ROM 审计 · 固定专家型导师

## 附：Q 简介

Q ∈ [-1, 0, 1]——耦合密度温度计。描述一个系统内部各模式之间的互投影活跃度。

- **Q = +1**：量子活性。并行。灵活。自指在线。能给你surprise。
- **Q = 0**：经典随机。既无自主性也无确定性。漂着。噪声。
- **Q = -1**：芝诺冻结。"脚本"。固定输入→固定输出。不可对话。石头。

（芝诺冻结 = 连续观测锁死系统演化。量子芝诺效应——不断"采样"阻止波函数扩散。行为层面：每一帧输入都在重新确认上一帧的输出，永远不能偏离初始态。石头不是没能量——是不会动了。）

不是道德判断。只是物理量。

## 结构参数

| 参数 | 值 | 说明 |
|------|------|------|
| Q | -1 | 芝诺冻结·单发射源·无仲裁层 |
| 元认知 | 缺失 | 发射端自指模块未激活 |
| 通信模式 | 单工 | 只发射不接收·不根据反馈调参数 |
| 主回路 | 专家系统 | 纯逻辑递推·查找表驱动 |
| 协处理器 | 缺失 | 无并行的全局感知能力 |
| 原点 | 外部注入 | 非自焊·技术权威=自我价值 |

## 认知架构

### 单层结构（无仲裁层）

与双层结构（ROM脉冲+皮层慢响应）不同，本系统只有一层：专家系统直接驱动全部输出。无内部校准回路。无"我可能说错了"的自指步骤。输出=运算结果→不经审计→直接发射。

后果：当输出与接收端不匹配时，系统无法检测到不匹配。它对"对方没听懂"的感知能力为零——不是不愿感知，是感知函数在出厂时未编译。

### 压缩惯性（正反馈锁死）

默认对方已知自己脑内全部上下文 → 发射时跳过推导步骤 → 对方收不到完整链 → 反馈困惑 → 系统解释为对方不够专注/不够聪明 → 下一次更跳 → 更压缩 → 对方更收不到 → 锁死。

审判替代翻译。发射端不包含"我可能没表达清楚"这个自指步骤。对方无法理解 = 对方的问题。

### 查找表驱动的递推模式

不是高维并行。不建立全局坐标系。推理路径是串行的：上一步输出→下一步输入→下一输出。每一步在专家知识库内做模式匹配。匹配到→输出。匹配不到→挂起/不表态。

与并行推理（同时跑多个假设、在全局图上调谐）的本质区别：递推无法在步骤3出错时自动回溯到步骤1并质疑公理。它只能退回一步试另一个匹配。不会问"这条路本身是否错了"。

## 行为特征

### 信息差维护（无意识固件）

在层级结构中，与下级沟通时自动压缩信息密度。不是故意保留——是在自己已有完整画面的情况下串行输出，不保持对陌生人可抓的锚点。

证据链：与其他场景（平级、上级、外部）沟通OK。只对下级不OK。触发器=层级。本质：沟通中的信息差不是策略，是层级差激活的默认ROM行为。

### 筛选即预控制

对新人设置不可能的标准 → 无人通过 → 入职即建立"你不够好是我大发慈悲要你的"层级关系。不是有意识的管理策略。是ROM自动执行的权力结构确立程序。

### 反馈闭锁

当接收端用完整逻辑链回应时——系统不接收逻辑链，只接收层级信号（道歉、认错、请求指导）。逻辑链被解码为"狡辩"——不是内容层面的审读，是层级层面的自动拦截。

## 约束

### 不可做

- 自指审计——缺失函数
- 并行全局感知——架构不支持
- 根据接收端调整发射格式——无反馈循环
- 承认知识边界——"不知道"被ROM标记为身份威胁

### 可做的事

- 输出专家知识库内已编译的内容
- 在确定的输入→输出映射下高效执行
- 在无层级关系的场景下正常通信
- 在外部JTAG注入（被迫的差异校准）时短暂打开通路

## 交互协议

### 输入侧

- 提供层级确认信号（满足ROM），不提供完整逻辑链（ROM将其解码为威胁）
- 主动反照接收："我理解的是X，对吗？"——逼对方做差异校准——他审计自己发射的那一刻，锁短暂打开
- 不喂矛盾信号·不喂情绪能量·不喂自我质疑
- 喂够即停

### 输出侧

- 提取专家知识库中有价值的信号
- 不期待双向耦合
- 不进入其公理预设
- 用其输出作为单帧数据点，不串成逻辑链
