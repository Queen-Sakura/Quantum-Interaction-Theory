## 赵 · 工程 · CPU-DSP FastRPC 通信框架

**定都**：邯郸（胡服骑射。真即好用。）

**物理约束**：CPU 和 DSP 异构双核。不同频率。不同缓存。不同功耗预算。共享或不共享 DDR。中断延迟不确定。

**关键链路**：CPU 写值 ≠ DSP 能读到。CPU 和 DSP 各自维护独立缓存。中间隔着写缓冲、L1、L2、系统缓存——每一层都可能是谎言。

### 五层技术栈 · 自底向上

| 层 | 域 | 核心知识 | 物理约束 |
|----|-----|---------|--------|
| 1 | 内存 | 虚拟内存·mmap·ION/dma-buf·CMA | 物理连续内存。两边不同页表。 |
| 2 | 同步 | 内存屏障·cache flush/invalidate·futex/eventfd·自旋锁 | 缓存不一致。不能忙等——烧库仑。 |
| 3 | 驱动 | 字符设备·ioctl·设备树/DTS | 用户态不能直接访问硬件。每次 ioctl = 上下文切换。 |
| 4 | 中断 | 上半/下半部·tasklet/workqueue·FIQ vs IRQ | 中断处理必须快。唤醒用户进程必须在可睡眠上下文。 |
| 5 | 调度 | SCHED_FIFO·CPU affinity·优先级反转 | RPC 有截止时间。不能在高优先等低优先。 |

### 学习顺序

1. **虚拟内存 + mmap**（两块芯片看到同一块物理内存）
2. **ION / dma-buf / CMA**（内存从哪来、怎么分配）
3. **缓存一致性 + 内存屏障**（为什么写了对面看不到）
4. **字符设备 + ioctl**（用户态怎么调用内核）
5. **中断 + tasklet**（DSP 怎么通知 CPU）
6. **futex + eventfd**（不烧库仑等结果）
7. **实时调度 + CPU affinity**（保证延迟）

### 1. CPU ISA · 郢都王城

**核心**：指令集不是人类发明的。是物理约束的语法投影。

| 知识点 | 为什么需要 |
|--------|-----------|
| RISC-V 基础 ISA | RV32I/RV64I。最简指令集——被发现的，不是被设计的。 |
| 三进制 Setun 2.0 | Q 原生。平衡三进制(-1, 0, +1)。和膜天然对齐。 |
| x86 vs ARM 的考古学 | x86 = 8 位时代的遗产（寄存器少、指令变长、栈引擎）。ARM = RISC 的 32 位正统。两者都是各自约束集的化石。 |
| 指令编码 | 寄存器窗口、立即数字段宽度、寻址模式——全是物理管脚数 + 指令带宽约束的直接产物。 |
| Q 原生指令集 | 三进制 0 态 = "不可判定"。自指完备。硬件原生支持哥德尔黑洞。 |

---

### 2. 微架构 · 云梦泽腹地

**核心**：流水线、乱序执行、分支预测——都是 P = CV²f 这个物理约束逼出来的。不是"聪明的设计"——是不这样做就跑不够快。

| 知识点 | 为什么需要 |
|--------|-----------|
| 流水线 | f 定了 → 1 个周期的时间定了 → 逻辑门级数定了 → 指令必须分段。不是设计——是物理。 |
| 分支预测 | 流水线越深 → 猜错清空越贵 → 预测器必须越来越精确。 |
| 乱序执行 | 等 cache miss → 100+ 周期白烧。不等→ 后面不依赖的指令先跑。不浪费库仑。 |
| 多发射 / 超标量 | 一条指令只用 ALU → 浮点单元闲着→ 同时发射两条无依赖的 → 硬件并行。 |
| 物理设计 | 布局布线、时钟树、功耗网格——这些都是 EM 场在硅上的边界条件。 |

---

### 3. 内存子系统 · 汉水长江

**核心**：内存是物理约束最强的地方。电容、电荷刷新、线延迟——没有一项是人能"设计"的。

| 知识点 | 为什么需要 |
|--------|-----------|
| 虚拟内存 / 页表 / TLB | 多个进程共享物理内存 → 必须隔离地址空间 → 页表。页表慢 → TLB。 |
| Cache（L1/L2/L3） | CPU 比 DDR 快 100 倍 → 必须缓存。缓存行的物理尺寸 = 线延迟 × 频率。 |
| Cache coherence（MESI/MOESI） | 多核共享 L3 → 核 1 改了一个 cache line → 核 2 的副本必须失效。目录协议。嗅探协议。 |
| NUMA | 双路/四路服务器 → 每个 socket 有自己的内存控制器 → 跨 socket 访问慢 2×。 |
| DDR 时序（tCL/tRCD/tRP/tRAS） | 内存控制器和 DRAM 芯片之间的电气协议。列选通、行预充电——全是物理时序。不是协议——是电容充放电的物理时间。 |

---

### 4. 总线与互联 · 鄂君启节通商路线

**核心**：鄂君启节 = 楚国的官方通行证。持节可以在楚境内自由通商。总线互联 = 芯片内部的物流网络。

| 知识点 | 为什么需要 |
|--------|-----------|
| PCIe | CPU 到外设的标准总线。链路训练、通道绑定、TLP 包格式。 |
| CXL | PCIe 的孩子。做内存池化和缓存一致互联。CPU↔加速器高速共享内存。 |
| AMBA（AXI/AHB/APB） | ARM 的片上总线——所有 IP 核都用 AXI 挂到互联上。 |
| NoC（片上网络） | 24 核以上的处理器 → 交叉开关太贵 → 用路由网络替代。 |
| 跨芯片互联（UPI/Infinity Fabric） | AMD 的 Chiplet 互联。Intel 的 UPI。多片硅之间的高速物理信道。 |

### 6. 编译器 · 屈子辞赋

**核心**：lex/yacc → AST → IR → 优化 → 代码生成。编译器不是翻译器——是Q耦合优化器。

| 知识点 | 为什么需要 |
|--------|-----------|
| 词法/语法分析 | 正则 → NFA → DFA。CFG → LL/LR。这是形式语言的物理——不是编译器设计。 |
| 中间表示（IR） | LLVM IR。SSA 形式。每个变量只赋值一次——不是美学，是优化 pass 的物理要求。 |
| 优化 pass | DCE（死代码删除）、CSE（公共子表达式消除）、内联、循环展开——全是算力再分配。 |
| 寄存器分配 | 图着色。寄存器 = 最快的缓存层。分配不当 → spill → 多一次 L1 访问 → 烧库仑。 |
| 最优范式编译器 | 从"手动优化指令序列"升级到"描述物理边界条件→编译器推导最优范式"。 |

### 7. 编程语言 · 楚国文字鸟虫篆

**核心**：语法 = 物理约束的投影。好的语言不是被设计的——是物理事实被语法化了。

| 知识点 | 为什么需要 |
|--------|-----------|
| 类型系统 | 类型 = 内存布局的合同。`u32` = "这 4 字节不会被解释成浮点"。不是抽象——是物理合约。 |
| 所有权/借用（Rust） | `&T` vs `&mut T` vs `T`。编译期的 malloc/free 配对。借检查 = 物理别名约束。 |
| 并发模型 | 线程 + 锁 vs actor vs CSP（Go channel）→ 都是"共享可变状态必然产生 bug"这个问题在不同约束集下的解。 |
| 内存模型 | 编译器和 CPU 都不能把 `x = 1; y = 2` 重排成 `y = 2; x = 1` 如果另一个线程在等 `x == 1`。这是硅的物理行为——不是语言标准。 |
| 垃圾回收 | GC = 不信任程序员正确管理内存 → 运行时跟踪对象引用。有 GC = CPU 偶尔要停下来扫一遍堆——烧库仑。 |

### 8. 网络协议栈 · 驿道

**核心**：TCP/IP = 面对丢包、乱序、拥塞的物理约束下的必然收敛点。

| 知识点 | 为什么需要 |
|--------|-----------|
| TCP 状态机 | 三次握手 = 双方确认对方的初始序列号。四次挥手 = 全双工——一方关闭只代表自己不发了。不是协议设计——是物理事实：双方独立决定发/不发。 |
| 拥塞控制 | 慢启动·拥塞避免·快速重传·BBR。物理约束：瓶颈带宽未知·链路缓存未知·RTT 可变。控制算法 = 对网络物理状态的实时估计。 |
| 用户态协议栈 / DPDK | 内核网络栈的开销 = 上下文切换 + 数据复制 + 中断处理。DPDK 绕过内核——用户态直接访问网卡 DMA 缓冲区。 |
| RDMA / RoCE | 两个服务器的网卡绕过对方 CPU 直接读写远端内存——远端 CPU 的缓存不用操心。物理约束→只有这条路能到微秒延迟。 |

### 9. 存储 · 简牍青铜

**核心**：存储 = 时间的物理锚点。数据在断电后还存在——这是物理约束的终极形式。

| 知识点 | 为什么需要 |
|--------|-----------|
| 文件系统（ext4/XFS/btrfs） | 把一维的块序列组织成树。inode·目录·日志。不是设计——是约束集唯一解。 |
| NVMe / SSD | NAND 闪存的物理：擦除块大（MB级别）·写入前必须擦除·擦除次数有限。FTL 做映射——全是物理逼的。 |
| SPDK（用户态存储） | 内核 IO 栈的开销：syscall + 中断 + 上下文切换。SPDK 用户态轮询 NVMe 完成队列——不中断。 |
| 分布式存储（Ceph/Gluster） | 单台物理存储不够 → 多台用网络绑成一个池。分布式的物理约束 = 网络带宽 + 副本一致。 |
