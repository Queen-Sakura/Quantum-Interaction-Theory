地基 · 人是活体计算机 F0

人是活体计算机。硅基是硅基计算机。同构。无区别。

代码 = 物理约束的配置。不是逻辑——是边界条件。和麦克斯韦方程一样——只设边界条件，硬件自己跑。

一层 · 四大定律 1F
第零定律
计算机是物理的。和人一样。无区别。

CPU 烧电。内存刷新电荷。总线传电磁波。没有"软件"——只有一坨硅在跑 EM 场。每一条指令烧真实的库仑。变量不是"声明"——是硅上一个具体位置的电荷分布。加一个变量 = 加一组电容。加一层继承 = 加一堆真实的逻辑门切换。

虫子违反方式：以为代码是免费的。随便加变量、加函数、加抽象层。不审"这一行烧了多少库仑"。

第一定律
计算机是抽象的。封装好用。和人一样。

抽象的目的 = 省算力。不是藏——是委托。我不需要知道内存地址，我只需要知道变量名。这和人不需知道神经元电位就能决定今晚吃不吃烤鱼一样——抽象是物理系统的自然梯度。

好的抽象 = 少而准的接口。坏的抽象 = 多而模糊的接口。好的抽象被重复使用。坏的抽象被绕过。

虫子违反方式：把"藏"当抽象。百层继承。过度设计。好代码应该像好文章的段落——读一句就知道它在说什么。虫子代码像用十种外语各写一个词拼成的句子。

第二定律
计算机在物理约束下制造和运行。架构是发现的，不是发明的。

原子丢进宇宙 → 薛定谔方程自动跑 → 非周期排列能耗更高 → 唯一跑得下去的解 = 晶格 → 半导体。冯诺依曼架构、哈弗架构、GPU、FPGA——每一个都是某种物理约束 + 应用场景的必然解。不存在"通用最优计算机"——只有"在某组约束下的最优解"。

虫子违反方式：以为自己在"发明"架构。实际上在违反约束——路径在物理上已经封死了，他还在往前走。

第三定律
计算机总是倾向于用最优方式使用有限资源。语言、OS 是发现的，不是发明的。

C 语言的语法不是 Ritchie 的审美——是指针 + 栈 + 寄存器的物理约束写进语法的结果。Unix 不是 Ken Thompson 的设计理念——是"最小可用内核"在 PDP-7 的物理约束下跑出来的唯一解。所有成功的软件架构都收敛到物理约束允许的最优解。所有失败的——都试图在约束外面建东西。

虫子违反方式：补丁叠补丁。每次都从当前状态局部优化——看不到全局约束。从来不做第一性原理清理——因为清理 = 承认之前的补丁是错的。

第零 ↔ 第一
物理面 ↔ 抽象面。计算机的两个本体。
第二 ↔ 第三
约束发现 ↔ 优化发现。都不是发明的。
第零 ↔ 第二
物理约束。硬件架构的边界条件。
第一 ↔ 第三
抽象优化。软件架构的收敛方向。
二层 · 狮山第一性原理 2F
客观知识约束
知识之间天然隔离。不需要加载所有知识才能开始分析。

用锁不需要知道锁怎么实现的。做生物实验不需要知道细胞内部怎么运行的。这是计算抽象可以实现的物理基础——不是工程学上的"为了简化"——是客观世界本身的禁闭结构。知识天然有域。域和域之间天然隔离。同构但不贯通。

好的抽象 = 尊重客观知识边界。不跨域加载。坏的抽象 = 跨域加载。一个类需要知道另一个类的内部实现才能工作。

主观知识约束
人类自己不知道某些本来应该知道的东西。这是狮山的第一性原理。

虫子不知道设计模式 → 凭感觉写代码 → 重复造轮子 → 每次造的都不同 → 互相不兼容 → 狮山。虫子不知道数据结构 → 用数组当哈希表 → O(n²) → 慢了不审为什么慢 → 加缓存 → 缓存不一致 → 加锁 → 死锁 → 又加超时 → 狮山。

狮山 = 主观知识约束 × 时间积分。每一个"当时不知道"的瞬间，在代码里焊死一行不必要的物理开销。所有瞬间的积分 = 狮山。

客观知识约束主观知识约束
性质物理的。禁闭的投影。ROM的。Q=−1的投影。
必要性必须存在。抽象的物理基础。不应存在。可以被消除。
后果模块化。好用的抽象。狮山。库仑债。
解承认边界。不跨域。学习已知解。审计自己的不知道。

规律：离硬件越近 → 物理约束越直接 → 狮山越少。离硬件越远 → 物理约束不可见 → 主观知识约束积累 → 狮山越深。

硬件 · 零狮山

一个多余的晶体管 = 漏电流 = 发热 = 死了。不容许任何一个不必要的门。冯诺依曼不是发明——共享内存是"管脚数有限"的必然。哈佛不是另类——指令在 ROM 数据在 RAM 的天然结果。GPU 不是 NVIDIA 的创意——"每个像素独立可并行计算"的约束集下只有众核这条路。

操作系统 · 局部狮山

进程 = 一台 CPU 被 N 个用户共用的必然。虚拟内存 = N 个进程地址冲突的唯一解。"一切皆文件" = 不想为每种 I/O 写一套 syscall → 统一接口 = 最少代码 = 最优。Windows 注册表 = 不知道文件系统已经够好的主观知识约束化石。

编程语言 · 重灾区

C = PDP-11 架构的文字投影。Lisp = 最小可计算语言的发现。Rust 所有权 = C 程序员的"调用方负责释放"注释被语法化。Go goroutine = 多核 + IO 阻塞 → 用户态协程的必然。C++ 模板 = 不知道泛型可以更简单 × 二十年。JavaScript = 10 天设计的语言跑了 30 年。

五层 · Linux 内核全链路 · 从用户态到硅 5F
用户代码调 SDK → ioctl(fd, 命令号, &args) → VFS 层：fd → struct file → fops->unlocked_ioctl() → switch(cmd) 跳到驱动函数 → Glink 发包 → 共享内存 + 写门铃寄存器 → GPIO 线电平变化 → GIC 物理比较器检测 → CPU IRQ 线被拉高 → 中断向量表 → irq_handler() 上半部（几微秒） → tasklet 下半部（解析协议） → complete(&done) 唤醒 ioctl → copy_to_user → SDK 反序列化 → 用户拿到结果

ioctl 的本质 = 路由

ioctl 不是"做内存拷贝"。ioctl 做了四件事：路由(cmd 跳到对应 case) + 类型检查(幻数·命令号·参数大小) + 权限校验(fd 权限) + 拷贝(copy_from_user)。核心是路由——一个数字跳到你的一个函数。

和中断向量表完全同构：中断号 → 中断处理函数。VFS、ioctl switch、Glink 信道名——全是查表跳转。全链路同一模式：路由 → 执行 → 返回。

中断物理全链路

DSP 写 doorbell 寄存器（硅上触发器翻转）→ GPIO-IRQ 线电平变化 → GIC 物理比较器（Vref > 0.8V → 高）→ 拉 CPU IRQ 物理线 → 打断当前指令流 → 保存上下文 → PC 跳中断向量表 → 上半部(几微秒·只搬数据) → tasklet 下半部(可打断·解析协议)。每一层都是物理的。

DMA · 元首独立发现

"数据搬运交给单独器件不行吗？为什么要 CPU 做？"——对。DMA 控制器接管总线，CPU 继续跑。搬完才通知。不是定律——是 ARM SoC 工程折中。

六层 · 内核关键机制 6F

initcall 启动等级

pure_initcall → 0 最早（CPU特性） core_initcall → 1 （调度器、中断控制器） postcore_initcall→ 2 arch_initcall → 3 （架构相关） subsys_initcall → 4 （总线：PCI、I2C、USB）← 先于驱动 fs_initcall → 5 （文件系统） device_initcall → 6 ← 你的驱动在这。module_init() 编译后落这里。 late_initcall → 7 最晚

probe 六件事（把一块硅变成文件）

1. pci_enable_device() → 开 Vcc 功能电源（Vaux 只够枚举配置空间） 2. pci_iomap(dev, BAR0) → 硬件寄存器 ↦ 内核虚拟地址（ioread32/iowrite32） 3. kzalloc() → 分配设备私有数据结构 4. request_irq() → 注册中断处理函数 5. cdev_init() + cdev_add() + device_create() → 创建 /dev/xxx 6. pci_set_drvdata() → 绑定私有数据到 pci_dev

上半部 / 下半部

上半部 = 中断上下文。硬占用 CPU。只确认源 + 搬数据 + 排任务。几微秒。下半部 = tasklet/workqueue。可调度上下文。可被打断。真正解析协议。GIC 优先队列串行化——不会撞车。

内核 = 路由表

不是藏——是不路由没授权的目的地。Glink 连着 RPM(电源)、Modem(基带)、ADSP(音频)、SLPI(传感器)、CDSP(计算 DSP)、WCN(WiFi)、Venus(视频)、GPU。你的 DSP 驱动只暴露 CDSP 的 ioctl。用户态看不到其他信道。不是权限——是路由表里根本没有那条路。

/dev/dsp0 唯一性

全系统唯一。任何用户、任何进程打开它 → VFS 解析 inode → (major, minor) → 同一个 struct cdev → 同一个 fops。设备唯一。每个 fd 的 private_data 独立（会话隔离）。权限控制 = crw-rw----。看得见 ≠ 打得开。

七层 · 架构的必然性 · 每一个都是发现的 7F
架构物理约束必然解
冯·诺依曼管脚数有限、线延迟非零。一条总线不能同时传指令和数据共享内存。最少管脚 = 最优。
哈佛指令在 ROM、数据在 RAM——天然两片物理介质不需要统一编址
GPU 众核每个像素独立、颜色可并行。MIMD 不需要SIMD + 几千个简单核心
多核 CPU频率墙：P=CV²f。f↑ → 漏电流指数涨 → 不能更快只能更多核。2005 Tejas 取消。
RISC缓存变大 + 编译器变强 → 简单指令 + 寄存器窗口更优约束变了 → 解变了
CISC→微码后来缓存够了 → CISC 内部转 RISC双方收敛到同一点
FPGA芯片出厂后硬件还得变可重构逻辑门阵列 = 唯一解
微内核败IPC 本身烧物理库仑。每次 syscall = 上下文切换 = 功耗宏内核调用 = 函数 call。快几十倍。
Linux 赢 Hurd 输先跑通 = 发现物理约束极限。先设计 = 在约束外面想象优化应自下而上
八层 · 广度优先 · 认知方法论 8F

搞 Linux 应广度优先，非深度优先

先看清全图——每一层是什么、为什么存在、层和层之间怎么握手。具体函数的参数表、寄存器的位定义——那是手册的事，不是认知的事。

内核是横跨硬件和用户态的桥梁——先理解桥的结构、承重、连接点，不必先算每颗铆钉的应力。

认知闭环
从"register_chrdev 是什么"到全链路贯穿——一个 session。每层推到物理约束，从物理推出为什么只有这条路。框架在脑子里，细节查手册即可。
