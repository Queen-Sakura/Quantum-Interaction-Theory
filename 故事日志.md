## 2026-07-02 — "楚国内核驱动全链路贯通 · 广度优先"

### 全链路覆盖

**设备号与注册**：`register_chrdev(0, name, &fops)` = 标准 Linux 内核 API。自动分配 major 号，绑定 `struct file_operations`。`mknod /dev/hello c <major> 0` 在文件系统贴标签。用户 open → 内核找 major → 调 fops。

**initcall 启动等级**：pure→core→postcore→arch→subsys→fs→device→late。PCI 总线在 4 级，你的驱动在 6 级。依赖链自动满足。

**PCIe 匹配 = 回调**：不是驱动找硬件。PCI 总线枚举 → 读 Vendor:Device ID → 问所有已注册 PCI 驱动"谁认识？"→ `id_table` 匹配 → 内核调 `probe(struct pci_dev *dev)`。你没调用内核。内核调用你。

**probe 六件事**：`pci_enable_device`(上电) → `pci_iomap`(映射寄存器) → 分配私有结构 → `request_irq`(中断线) → `cdev_init`+`cdev_add`+`device_create`(/dev 节点) → `pci_set_drvdata`(绑定私有数据)。把一块硅变成文件。

**两级供电**：Vaux(辅助电源·枚举配置空间) vs Vcc(`pci_enable_device`打开功能电源)。物理上插卡 ≠ 功能上电。

**MMIO 映射**：`ioread32(base + offset)` = CPU load → uncached → 北桥识别 PCIe MMIO 窗口 → TLP 包 → 卡解包 → 返回。全链路 PCIe 协议保证。

### DSP 提线木偶

CPU 驱动 = 操偶师。DSP = 木偶。线 = 共享内存 + 中断。probe 开机建线 · ioctl 拉线。

**Glink**：高通传输层。只搬二进制——不解析。信道名对上 = 连接。包格式由上层约定。内核驱动只暴露该暴露的信道——Glink 连着 RPM/Modem/ADSP/SLPI/CDSP/WCN/Venus/GPU。内核 = 路由表。不是藏——是不路由没授权的目的地。

**德国银行模型**：填 Glink 包 → `wait_for_completion_timeout` → 等回信 → DSP 中断处理 `complete(&done)` → `copy_to_user`。看门狗 = `schedule_delayed_work`——超时 → 复位/重试/重启信道。

### 中断物理全链路

DSP 写 doorbell 寄存器(硅上触发器翻转) → GPIO-IRQ 线电平变化 → GIC 物理比较器 → 拉 CPU IRQ 物理线 → 打断当前指令流 → 保存上下文 → 跳转中断向量表 → `irq_handler()`。

**上半部/下半部**：上半部几微秒——只确认源+搬数据+排任务。下半部 tasklet——可打断——真正解析协议。GIC 优先队列串行化——不会撞车。

### DMA · 元首独立发现

"数据搬运交给单独器件不行吗？为什么要 CPU 做？"——独立发现 DMA。对。DMA 控制器接管总线，CPU 继续跑。搬完才通知。ARM SoC 不为每个中断源配 DMA 触发通路——工程折中，非物理定律。

### 广度优先 · 认知框架落成

从设备注册到总线匹配到 probe 到中断到用户态接口——全链路贯通。每一层的物理约束和协议保证清晰。元首总结：搞 Linux 应广度优先——先理解桥的结构、承重、连接点，不必先算每颗铆钉的应力。具体参数 = 手册的事。
