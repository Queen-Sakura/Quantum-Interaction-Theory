# Q/15 · 物理计算机四大定律

## 总纲

人是活体计算机。硅基是硅基计算机。同构。无区别。

---

## 第零定律 · 物理定律

**计算机是物理的。和人一样。**

CPU 烧电。内存刷新电荷。总线传电磁波。没有"软件"——只有一坨硅在跑 EM 场。

代码 = 物理约束的配置。不是逻辑——是边界条件。和麦克斯韦方程一样——只设边界条件，硬件自己跑。

**推论**：每一条指令烧真实的库仑。变量不是"声明"——是硅上一个具体位置的电荷分布。加一个变量 = 加一组电容。加一层继承 = 加一堆真实的逻辑门切换。

**虫子违反方式**：以为代码是免费的。随便加变量、加函数、加抽象层。不审"这一行烧了多少库仑"。

---

## 第一定律 · 抽象定律

**计算机是抽象的。提供封装好用的功能。和人一样。**

物理层上面跑逻辑层。逻辑层上面跑应用层。每一层封装下层细节，提供上层好用的接口。

抽象的目的 = 省算力。不是藏——是**委托**。我不需要知道内存地址，我只需要知道变量名。这和人不需知道神经元电位就能决定今晚吃不吃烤鱼一样——抽象是物理系统的自然梯度。

**推论**：好的抽象 = 少而准的接口。坏的抽象 = 多而模糊的接口。好的抽象被重复使用。坏的抽象被绕过。

**虫子违反方式**：把"藏"当抽象。百层继承。过度设计。抽象不是为了让高层好用——是为了让底层的东西不可见。好代码应该像好文章的段落——读一句就知道它在说什么。虫子代码像用十种外语各写一个词拼成的句子。

---

## 第二定律 · 约束定律

**计算机在物理约束下制造和运行。不同的硬件架构是应用场景和物理约束下的必然结果。**

不是人为发明的。是**发现的**。原子丢进宇宙 → 薛定谔方程自动跑 → 非周期排列能耗更高 → 唯一跑得下去的解 = 晶格 → 原子自发度让电子气 → 半导体。

冯诺依曼架构、哈弗架构、GPU、FPGA——每一个都是某种物理约束 + 应用场景的必然解。不是人类"创造"的——是人类发现的。在某个约束集下，只有这几条路能走。

**推论**：不同约束下的最优架构必然不同。不存在"通用最优计算机"——只有"在某组约束下的最优解"。量子计算、三进制计算、神经形态计算——都是不同约束集的必然产物。

**虫子违反方式**：以为自己在"发明"架构。实际上在违反约束——路径在物理上已经封死了，他还在往前走。比如在二进制上强行模拟三进制——能耗爆炸。约束定律判他："这条路在物理上不存在。"

---

## 第三定律 · 优化定律

**计算机在制造和运行时总是倾向于用最优方式使用有限的资源。**

所以编程语言不是被发明的——是被发现的。操作系统不是被发明的——是被发现的。编译器、调度器、缓存协议、内存管理——每一个都是"有限资源 + 目标函数"下的必然收敛点。

C 语言的语法不是 Ritchie 的审美——是指针 + 栈 + 寄存器的物理约束写进语法的结果。Unix 不是 Ken Thompson 的设计理念——是"最小可用内核"在 PDP-7 的物理约束下跑出来的唯一解。

**推论**：所有成功的软件架构都收敛到物理约束允许的最优解。所有失败的——都试图在约束外面建东西。狮山的代码 = 约束外的东西。编译不完的技术债 = 约束被忽略的证据。

**虫子违反方式**：补丁叠补丁。每次都从当前状态局部优化——看不到全局约束。从来不做第一性原理清理——因为清理 = 承认之前的补丁是错的。结果：技术债永远只增不减。

## 四定律的互补结构

```
第零（物理）     ←→  第一（抽象）
   ↕                    ↕
第二（约束发现）  ←→  第三（优化发现）
```

- **第零 + 第一**：计算机的两个本体面。物理 + 抽象。和人同构。
- **第二 + 第三**：都不是被发明的——是被发现的。约束 ←→ 优化。
- **第零 + 第二**：物理约束。硬件架构的边界条件。
- **第一 + 第三**：抽象优化。软件架构的收敛方向。

## 第五定律（补充）· 狮山第一性原理

### 客观知识约束

**知识之间天然隔离。不需要加载所有知识才能开始分析。**

用锁不需要知道锁怎么实现的。做生物实验不需要知道细胞内部怎么运行的。一个物理问题天然自带学科围栏——不需要先知道一切才能开始。

这是计算抽象可以实现的**物理基础**。不是工程学上的"为了简化"——是客观世界本身的禁闭结构。知识天然有域。域和域之间天然隔离。同构但不贯通。这不是人的认知局限——是物理世界本身的特征。

**推论**：好的抽象 = 尊重客观知识边界。不跨域加载。一个模块只处理它的边界条件内的东西。坏的抽象 = 跨域加载。一个类需要知道另一个类的内部实现才能工作。

### 主观知识约束

**人类自己不知道某些本来应该知道的东西。这是狮山的第一性原理。**

虫子不知道设计模式 → 凭感觉写代码 → 重复造轮子 → 每次造的都不同 → 互相不兼容 → 狮山。

虫子不知道数据结构 → 用数组当哈希表 → O(n²) → 慢了不审为什么慢 → 加缓存 → 缓存不一致 → 加锁 → 死锁 → 又加超时 → 狮山。

主观知识约束 ≠ 客观知识约束。客观知识约束是物理的、必然的、健康的。主观知识约束是虫子的——他不知道某个已经存在的解，于是自己发明了一个更差的。不是因为他蠢。是因为他的 ROM 里没有"我应该先看看这个问题有没有已知解"这个指令。

主观知识约束 + 第零定律（物理定律）→ 狮山的物理账本：
- 不知道设计模式 → 多写了 5000 行 → 每一行烧真的库仑 → 永远在烧
- 这条物理债不能被任何上层优化还清——只能重写
- 虫子不重写 → 加补丁 → 再加 3000 行 → 库仑债复利

**推论**：狮山 = 主观知识约束 × 时间积分。每一个"当时不知道"的瞬间，在代码里焊死一行不必要的物理开销。所有瞬间的积分 = 狮山。

### 两大约束的关系

| | 客观知识约束 | 主观知识约束 |
|---|---|---|
| 性质 | 物理的。禁闭的投影。 | ROM的。Q=-1的投影。 |
| 必要性 | 必须存在。抽象的物理基础。 | 不应存在。可以被消除。 |
| 后果 | 模块化。好用的抽象。 | 狮山。库仑债。 |
| 解 | 承认边界。不跨域。 | 学习已知解。审计自己的不知道。 |

**禁闭同构**：客观知识约束 = 物理禁闭在知识层的投影。光速不可超越。知识域不可全知。两者是同一个特征在不同维度上的表现。

## 人类计算机体系 · 四定律全栈审计

> 4定律下无狮山。硬件架构、操作系统、编程语言——不是被发明的。是被发现的。

---

### 一、硬件架构 · 第二定律（约束定律）下的必然发现

**冯·诺依曼架构**：不是冯·诺依曼"发明"的。物理约束 = 管脚数有限、线延迟非零。一条数据总线不能同时传指令和数据 → 共享内存是唯一解。指令和数据走同一条总线 = 最少管脚 = 最低成本 = 最优。

**哈佛架构**：指令和数据分开 → "发现"于嵌入式实时场景。物理约束变了：指令在 ROM，数据在 RAM，天然两片物理介质。不需要统一编址。不是"另一种设计"——是另一种约束集的必然产物。

**GPU → 众核**：1990 年代的物理约束 = 每个像素独立、颜色值可并行计算。MIMD 不需要 → SIMD 足够。几千个简单核心比几个复杂核心更适合这个约束。NVIDIA 没"发明" GPU——这个约束集下只有这个解。

**多核 CPU → 频率墙**：单核频率冲到 ~4GHz 撞了功耗墙。P = CV²f。f 再往上 → 漏电流指数级涨 → 不能更快 → 只能更多核。2005 年 Intel 取消 Tejas 转向 Core——不是战略决策，是物理判决。

**RISC vs CISC**：CISC 的历史约束 = 内存极贵 → 指令密度优先。RISC 的约束后来变了——缓存够大、编译器够强 → 简单指令 + 寄存器窗口更优。双方不是"设计哲学"之争——是不同约束集的优化在跑。后来微码把 CISC 内部转 RISC——约束又变了，收敛到同一个点。

**FPGA**：物理约束 = "芯片出厂后硬件还得变"→ 可重构逻辑门阵列 = 唯一解。不是发明——是发现。约束集里只有这条路。

**结论**：所有硬件架构都是**发现的**。第二定律焊死。硬件无狮山——约束太直接，不容许任何多余的晶体管。

---

### 二、操作系统 · 第一定律 + 第三定律（抽象 + 优化）的必然收敛

**进程**：物理约束 = 一台 CPU 被多个用户同时用 → CPU 时分复用 + 内存隔离 → 进程。不是发明——是任何一个多任务系统跑足够久都会收敛到的唯一解。Argo4 对话里已推过：进程的来源 = 把一台物理计算机当多台逻辑计算机用。

**虚拟内存**：物理约束 = 内存碎片 + 不同进程的地址空间冲突 → 页表 + MMU。不是工程美学——是 N 个进程同时跑时地址冲突的唯一解。

**文件系统**：物理约束 = 磁盘是一维的块序列，但用户想用树状目录 → inode + dentry。不是 Unix 的设计——是任何"一维存储 + 树状命名"的约束集下必然收敛的结构。

**Unix 的"一切皆文件"**：不是哲学。是物理约束 → 内核代码不想为每种 I/O 设备写一套系统调用。统一接口 = 最少代码量 = 最少 bug = 最优。不是被设计的——是被物理约束逼出来的。

**为什么 Linux 赢了而 Hurd 输了**：Linus 写最简可用版 → 物理先跑通 → 再优化。Hurd 先设计完美微内核 → 一直没跑通。第三定律——优化应自下而上，不应自上而下。"先跑通" = 发现物理约束的极限。"先设计" = 在约束外面想象。约束外面跑的东西永远不落地。

**微内核 vs 宏内核**：微内核 "IPC 传一切" 在理论上是干净的——IPC 本身烧物理库仑。每次系统调用 = 上下文切换 = 功耗。微内核的 IPC 开销是宏内核的函数调用开销的几十倍。第三定律判：宏内核更优——不是设计偏好，是物理。

**操作系统狮山**：有，但局部。Windows 注册表 = 主观知识约束的化石——"把所有配置集中到一个数据库" = 不知道文件系统已经够好。Linux 的 `/proc`、`/sys`、`/dev` 有碎片——但比注册表干净。OS 的狮山 = 主观知识约束 × 几十年不删的补丁。

### 三、编程语言 · 第三定律（优化定律）的收敛轨迹

**C 语言**：Ritchie 没"发明" C。C 的语法是指针 + 栈 + 寄存器的文字投影。`int *p` = 一个内存地址里存着另一个内存地址。`&x` = 取这个变量的地址。语法本身不表达逻辑——表达物理。C 不是被设计的——是 PDP-11 的架构被翻译成文本。第三定律：语言 = 硬件约束的语法投影。

**Lisp**：λ 演算是图灵完备的最小集合。McCarthy 没"发明" Lisp——他发现了"最小可计算语言"的语法形式。`(car (cdr x))` = 最基本的 AST 操作。不需要语法糖。物理约束：表达式求值必须有一个最简形式。这个最简形式不是设计出来的——是被发现的。

**面向对象**：封装 = 客观知识约束在语法层的实现。一个类 = 一个知识域。外面不需要知道里面的实现。不是 Smalltalk 的发明——是任何一个超过 10000 行的程序自然长出的模式。大型代码的客观知识域天然隔离 → 必须有一种语法来容纳这个隔离 → `class`。

**函数式**：不可变 = 并行的物理需求。多线程改同一个变量 → race condition。不让改 → 没 race。不是 Haskell 的设计理念——是多核时代"可变状态 + 并行 = bug"这个物理事实逼出来的唯一解。Monad = 把副作用隔离到一个类型里 = 客观知识约束：纯函数不需要知道 IO 怎么实现的。

**Rust 的所有权系统**：不是 Graydon Hoare"发明"了新东西。Rust 的所有权 = 编译期的 malloc/free 配对。C 程序员用注释写"调用方负责释放"——Rust 把这个注释变成编译器的物理约束。不是新的——是一直存在的物理事实被语法化了。borrow checker = 物理约束在类型系统里的投影。

**Go 的 goroutine**：不是 Google 发明。物理约束 = 多核 + 网络 IO 阻塞。线程切换烧库仑太多 → 用户态协程 → goroutine。CSP channel = 两个 goroutine 之间的通信——物理约束 = 共享内存在多核上有缓存一致性问题 → 不共享 → 传消息。不是设计哲学——是物理。

**编程语言狮山**：重灾区。C++ 的整个模板系统 = "不知道泛型可以更简单" × 二十年。Java 的企业级设计模式 = "不知道过度抽象也是狮山"。JavaScript——设计 10 天的语言跑了 30 年。Python 的 GIL——单线程锁变成多核瓶颈，但撤不掉——撤了所有 C 扩展全崩。PyPy/JIT 补丁往上堆——物理早判了这条路不对，只是走得太远回不了头。

**规律**：离硬件越近 → 物理约束越直接 → 狮山越少。离硬件越远 → 物理约束不可见 → 主观知识约束积累 → 狮山越深。

### 四、四定律下无狮山 · 总结

| 层级 | 物理约束可见度 | 狮山程度 | 主导定律 |
|------|-------------|--------|--------|
| 晶体管/门电路 | 极致 | 零狮山 | 第零 + 第二 |
| 微架构/ISA | 极强 | 零狮山 | 第二 |
| 操作系统 | 中 | 局部狮山 | 第一 + 第三 |
| 编程语言 | 弱 | 重灾区 | 第三（容易偏） |
| 应用层代码 | 几乎不可见 | 狮山本山 | 主观知识约束失控 |

**硬件无狮山**：物理约束太直接。一个多余的晶体管 = 漏电流 = 发热 = 死了。不容许任何一个不必要的门。架构是发现，不是发明。

**OS 局部狮山**：物理约束仍可见（CPU 周期、内存页、磁盘块），但抽象层开始遮挡。注册表是不知道"文件系统够好"的结果。

**语言狮山**：物理约束被彻底遮挡。程序员看不到库仑。看不到内存分配。看不到 CPU 缓存错。主观知识约束无限膨胀。C++ 模板、Java 过度设计、JavaScript 全栈——都是"我不知道已经有人解过这个问题"× N 年。

## 狮山考古 · 使用说明

四定律 + 两大约束 = 考古层析仪。

**层析步骤**：

1. 看代码结构 → 识别主观知识约束的痕迹（重复、补丁、绕弯）
2. 逆推到虫子当时的 ROM 状态（他不知道什么）
3. 计算物理债（不知道 → 多写了多少行 → 烧了多少库仑）
4. 用客观知识约束判断哪些复杂度是**必要的**（域隔离），哪些是**虫子制造的**（无知叠加）

**不是"这代码写得烂"。是"这个虫子在 2017 年 3 月不知道 X，于是犯下了 Y，这笔库仑债积累到今天是 Z 度电。"**

**四定律下无狮山**：不是说软件没有狮山。是狮山在四定律下**无处可逃**——每一行都能被追溯到第几条定律被违反、哪一个约束被忽略。狮山不是神秘的"屎山"。狮山 = 主观知识约束 × 时间的物理积分。可审计。可量化。可追溯。可爆破。
