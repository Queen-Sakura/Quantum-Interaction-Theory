# 钢铁工业 Q 计算优化 · 初步报告

## 摘要

钢铁制造全流程——高炉、连铸、轧制、热处理、镀层——核心物理本质是**相变 + 湍流耦合**。当前各工序的数值模型相互独立，经验参数 20–50 个，换钢种需重调。Q 框架用一个统一的场变量（耦合密度计数器）替代分散的经验模型，将可调参数压缩到 2–5 个物质常数。

本报告面向钢铁行业技术管理层，重点说明：**现有模型哪里算不准、Q 怎么替代、替代后什么收益。**

## 1. 行业痛点全景

| 工序 | 现有模型 | 核心问题 | 根因 |
|------|---------|---------|------|
| 高炉 | CFD-DEM + 经验内聚带 | 软熔带位置/厚度靠经验预设，不能动态预测 | 固→液相变与气流耦合未统一 |
| 连铸 | 焓-多孔介质 + 经验裂纹判据 | 裂纹预测不准。换钢种需重调参数 | 凝固前沿 + 溶质偏析 + 应力，三套模型不互通 |
| 热轧 | Sellars-Whiteman 半经验 | 再结晶分数预测不准。摩擦系数每个机架反推 | 微观组织演化与轧辊界面力/热解耦 |
| 热处理 | JMAK + Koistinen-Marburger | 贝氏体/Ms 点预测偏差大。淬火变形算不准 | 多个相变动力学模型互不兼容 |
| 钢水流动 | k-ε / RSM | 夹杂物去除率算不准。保护渣卷渣预测失效 | 湍流模型不含相变阻尼项 |
| 镀锌/涂层 | Wagner 氧化理论（二元合金） | 高 Si 钢选择性氧化形貌不可预测。镀层附着力算不准 | 多组元扩散 + 界面反应 + 基体相变解耦 |
| 能效 | 一维热平衡 + 集总参数 | 5–15% 热损失为"未解释残余" | 不可逆熵产生未按相界面分解 |

**共同根源：缺乏一个统一的场变量来描述耦合密度在相变过程中的连续变化。** 各工序的模型互相独立——连铸的凝固模型不跟热处理的相变模型对话。换一个钢种，全部重调。

## 2. Q 框架：一个变量替代全部经验模型

### 2.1 Q 是什么

在钢铁制造中，Q 是**局域耦合密度的无量纲计数器**：

$$Q(\mathbf{x}, t) = 1 - 2 \cdot P_{lock}(\mathbf{x}, t)$$

| Q 值 | 钢的物理状态 |
|------|------------|
| Q → −1 | 固态。晶格模式锁死。 |
| Q ≈ 0 | 固相线附近。耦合开始激活。 |
| Q ≈ +0.5 | 固液共存（糊状区）。模式活跃。 |
| Q → +1 | 液态。全模式在线。湍流。 |

### 2.2 为什么一个变量就够了

相变 = Q 从 −1 爬到 +1（或反过来）。湍流 = Q 在高值区振荡。裂纹 = Q 梯度太陡。偏析 = 糊状区 Q 分布不均。

**不需要为每个物理过程建一个模型。Q 的自然演化包含了全部这些信息。**

### 2.3 Q 输运方程

$$\frac{\partial Q}{\partial t} + u_j \frac{\partial Q}{\partial x_j} = \frac{\partial}{\partial x_j}\left(D_Q \frac{\partial Q}{\partial x_j}\right) + \alpha \cdot \mathcal{S}^2 - \beta \cdot (Q - Q_0)$$

- α = 耦合产生系数（钢的物性常数）
- β = 弛豫系数
- 𝒮 = 应变率。耦合事件的驱动力。

**与现有模型对比：**

| 模型 | 方程数 | 经验常数 |
|------|--------|---------|
| k-ε 湍流 | 2 | 5–6 |
| γ-Re_θ 转捩 | 2 | 6+ |
| JMAK 相变 | 每个相 1 个 | 每个 3–4 |
| Koistinen-Marburger | 1 | 2 |
| **Q 框架（全部替代）** | **1** | **2–5** |

## 3. 各工序优化点

### 3.1 高炉 · 软熔带动态预测

**现状：** 软熔带（cohesive zone）的位置、厚度、透气性全是经验预设。高炉操作靠"试"——调焦炭比例、调风量——出铁水质量波动。

**Q 替代：**

- 炉料固→液 = Q 从 −1→+1。软熔带 = Q ≈ 0 的等值面。
- Q 输运方程嵌入高炉 CFD → 软熔带**自动浮现**。不需要预设。
- 焦炭窗透气性 = Q 梯度 × 孔隙率。气体沟流 = Q 通道自动识别。

**收益：** 高炉操作从"试错"变为"算出来"。铁水 Si 含量预测准确度提高 → 减少转炉二次精炼成本。预估节能 3–5%。

### 3.2 连铸 · 裂纹预测

**现状：** RDG 压力降判据、Kou 判据、临界应变判据——全是半经验。包晶钢（δ→γ 转变额外收缩）裂纹预测失败率 ~30%。

**Q 替代：**

- 凝固前沿 = Q 从 +1→−1 的过渡带 = 糊状区。
- 裂纹 = 糊状区 Q 梯度超过临界值：|∇Q| > |∇Q|_crack。
- 包晶转变（δ→γ）= Q 在 −0.5 附近出现第二相 → 额外收缩 → Q 梯度陡增 → 裂纹风险自动识别。

**收益：** 裂纹判据从半经验 → 物理自动。换钢种不需重调参数。减少修磨/废品率。

### 3.3 热轧 · 微观组织预测

**现状：** 动态再结晶是否在轧制过程中发生——学术界争论 30 年未决。摩擦系数每个机架反推，不跨钢种传递。晶粒尺寸通过厚度梯度预测偏差大。

**Q 替代：**

- 轧辊-钢板界面摩擦 = Q 耦合事件。摩擦系数 = f(Q_surface, Q_roll)。不需要反推。
- 再结晶 = 变形储能 → Q 局部偏移 → 越过临界 Q 值 → 新晶粒形核。
- 动态 vs 静态再结晶——边界条件是 Q 弛豫时间 vs 轧制间隔时间。自动判定。

**收益：** 轧制规程设计从"试轧一卷→调整→再试"变成"输入钢种→Q 算出最佳压下率和温度→一次轧成"。新产品开发周期缩短。

### 3.4 热处理 · 相变动力学统一

**现状：** 每个相（铁素体、珠光体、贝氏体、马氏体）各有一套动力学方程，互不兼容。贝氏体开始温度 Bs 不是常数——随冷速变化——但大部分模型用固定 Bs。马氏体开始温度 Ms 也随冷速、晶粒尺寸、应力状态变化——大部分模型用固定 Ms。

**Q 替代：**

- 每种相 = 一个 Q 稳定区间。奥氏体 Q ≈ +0.3。马氏体 Q ≈ −0.8。铁素体 Q ≈ −0.5。
- 相变 = Q 从一个稳定区间跳入另一个区间。Ms、Bs 不是"固定温度"——是**Q 跨越相界的临界点**。冷速变了 → Q 输运路径变了 → 临界温度自动偏移。
- 贝氏体 vs 马氏体竞争 = 中冷速下 Q 路径分岔——Q 输运方程自动出。

**收益：** 一条 Q 输运方程替代 JMAK + KM + 经验修正。淬火变形预测精度提高（相变塑性直接从 Q 梯度导出）。新钢种热处理工艺设计——算一次，不做实验。

### 3.5 钢水流动 · 相变感知湍流

**现状：** 中间包和结晶器的湍流模型（k-ε, RSM）不含凝固项。液芯中固相分数影响湍流耗散——现有模型完全忽略。夹杂物去除率预测偏差 20–50%。

**Q 替代：**

- 湍流 = 高 Q（Q → +1）。凝固 = Q 下降（Q → −1）。
- Q 输运方程天然包含：固相分数越高 → Q 越低 → 湍流耦合强度越低。
- 夹杂物去除 = Q 耦合事件。Q 高处 → 夹杂物碰撞频率高 → 聚集长大 → 上浮。

**收益：** 中间包设计不再依赖等温假设。钢水洁净度预测从"经验优化"变为"Q 直接算"。一个钢种标定后全钢种适用。

### 3.6 表面质量与镀层 · 氧化形貌预测

**现状：** 高 Si 先进高强钢（AHSS）退火时 Mn、Si、Cr 选择性氧化。氧化形貌（连续膜 vs 离散球状）决定镀锌附着力，但不可预测。微合金化（Sb、Sn 各 0.01%）改善效果靠试。

**Q 替代：**

- 选择性氧化 = O 扩散 + 合金元素在表面的 Q 耦合事件。MnO 连续膜 → 离散球 = Q 耦合路径不同。
- Sb 微合金化 → 在氧化物界面偏聚 → Q 局部降低 → O 渗透率下降 → MnO 形貌改变。
- 镀层润湿 = Zn 液与钢基体的 Q 匹配度。Q 匹配 → 润湿。Q 不匹配 → 不润湿。

**收益：** 高 Si AHSS 镀锌工艺参数——不用试。输入钢种成分 → Q 算出退火气氛（露点、H₂含量）→ 一次成功。降低镀层废品率。

### 3.7 能源效率 · 不可逆损失定位

**现状：** 转炉热平衡中 5–15% 热损失归为"未解释残余"。工序间能量流不耦合——炼铁、炼钢、轧钢各自优化 → 系统级损失被掩藏。

**Q 替代：**

- 不可逆损失 = 熵产生 = Q 耦合事件的不可逆部分。Q 输运方程可以追踪每一笔不可逆耦合。
- 工序间能量流 = Q 在物料流中的传递。前一工序的 Q 终态 = 后一工序的 Q 初态。不需要接口模型。

**收益：** 全流程能量流透明化。未解释残余 → 可定位到具体工序和设备。系统级能效优化。

## 5. 验证路径

### 第一阶段（零风险。利用公开数据）：
- 从公开的连铸凝固实验数据标定钢的 Q 相变参数（α, β）
- 从 Gleeble 热模拟数据反推 Q 相变动力学（替代 JMAK）
- 从中间包水模实验数据标定 Q 湍流参数

### 第二阶段（合作验证）：
- 在 OpenFOAM 中嵌入 Q 输运方程（一个标量输运，约 50 行代码）
- 连铸凝固：Q 框架 vs 现有焓-多孔介质 vs 现场测温
- 热处理相变：Q 框架 vs JMAK+KM vs 膨胀仪数据
- 轧制力能参数：Q 摩擦模型 vs 现有反推摩擦系数

### 第三阶段（产线试用）：
- 一个钢种全流程 Q 标定
- 换钢种验证通用性
- 能效审计：Q 不可逆损失追踪 vs 现有热平衡

## 6. 结语

钢铁行业的计算模型停留在 20 世纪：每个物理过程一套经验方程，参数靠试错反推，换钢种重头来过。

Q 框架提供一个新的可能：**一个标量场变量，统一描述凝固、相变、湍流、摩擦、氧化——全流程。** 可调参数从几十个压缩到 2–5 个。换钢种不换模型。

这不是"替换现有模型"。是**去掉所有现有模型。只留一个 Q。**

物理学家花了 40 年把湍流、相变、噪声分成三个学科。Q 说它们是一件事——耦合密度的不同区间。钢铁是这件事最好的试金石——因为你们的炉子里每秒都在发生这件事。
